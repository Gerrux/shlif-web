# Рефактор интерфейса под дизайн-систему «Шлиф»

**Дата:** 2026-07-04
**Референс:** `Дизайн-система Шлиф.dc.html` (полная спецификация + канонический промпт)

## Цель

Привести существующий фронтенд (Next.js) в соответствие с дизайн-системой «Шлиф»:
холодная база, единственный латунный акцент, фазовые цвета строго по смыслу, тёмная сцена
шлифа, моноширинные числа, тонкие границы, line-иконки вместо эмодзи. Без изменений бэкенда
и без выдуманных данных (YAGNI: очередь образцов, таблица метрик, карта уверенности — не
добавляем, т.к. соответствующих данных в API пока нет).

Решения (подтверждено пользователем):
- **Объём:** полный рефактор существующих экранов (загрузка → анализ → вердикт → корректор).
- **Тема:** обе темы (светлая/тёмная) + переключатель.

## Что меняем

### Основа
- `app/globals.css` — портируем полный `<style>` дизайн-системы: `:root` токены +
  `[data-theme="dark"]`, базовые стили, все классы компонентов, которые использует приложение.
  Удаляем мёртвые Streamlit-селекторы (`.stApp`, `.block-container`, `[data-testid=...]`).
- `app/layout.tsx` — `<link>` на IBM Plex (preconnect), pre-paint скрипт темы (без мигания),
  `suppressHydrationWarning` на `<html>`.
- `components/icons.tsx` (новый) — inline line-иконки stroke 1.8: гексагон-лого, upload, pencil,
  save, undo, redo, sun, moon, alert. Заменяют эмодзи `◈ ✎ ↶ ↷ 💾`.
- `components/ThemeToggle.tsx` (новый) — контрол `.theme-toggle` (light/dark), сохранение в
  `localStorage`.

### Экраны
- `app/page.tsx`:
  - топбар: SVG-гексагон лого + eyebrow-крошка + h1 слева, переключатель темы справа;
  - выбор режима: сегмент-контрол `.seg` (Крупный план / Панорама);
  - загрузка: `<label class="btn primary">` со скрытым `input[type=file]` + иконка upload;
  - статус задачи: `.status-badge` (queued / running / done / error);
  - кнопка правки масок: `.btn.ghost` + иконка pencil (без эмодзи);
  - тёмная сцена `.stage-box` с оформленными пустым и загрузочным состояниями.
- `components/corrector/Corrector.tsx`:
  - инструменты и слои → сегмент-контролы `.seg` (активный = текущий tool/layer);
  - кисть/порог → `.slider` + моноширинная подпись значения;
  - undo/redo/save → `.btn` с line-иконками.
  - **Логика рисования/canvas не меняется** — только оформление панели инструментов.
- Компоненты вердикта:
  - `SortCard.tsx` — цвета полос на токены фаз (ordinary→normal/зелёный, hard→fine/красный,
    talcose→talc/синий) вместо инлайновых `rgb()`;
  - `PhaseBars.tsx` — убрать no-op `{verdict.text ? "" : ""}`;
  - `VerdictPanel.tsx` — точка уверенности в подвале, выравнивание по `.verdict`.

## Принципы (из промпта ДС), которым следуем
- Цвет = смысл: латунь = бренд/действие/сульфиды; зелёный/красный/синий = только фазы.
- Сцена шлифа (`--stage`) тёмная в любой теме.
- Все числа (проценты, размеры, порог, F1, coord) — `--font-mono`, tabular-nums.
- Граница вместо тени; форма приборная (карточки r12, контролы r8, пилюли 999).
- Никаких эмодзи — только line-иконки stroke 1.8. Только IBM Plex Sans/Mono.

## Вне объёма (YAGNI)
- Каркас рабочей станции с сайдбаром и очередью образцов (нужен бэкенд-эндпоинт списка задач).
- Таблица метрик мкм², карта уверенности heatmap, масштабная линейка с реальными координатами —
  нет данных в API.
