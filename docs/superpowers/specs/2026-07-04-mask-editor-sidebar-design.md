# Редактор масок — таб-сайдбар + вид/слои-подсказки (по образцу hakaton_nornikel)

**Дата:** 2026-07-04
**Референс:** `../hakaton_nornikel/annotate_talc.py` (Streamlit-аннотатор, сайдбар `main()` строки 472-548)

## Цель

Поднять «Редактор масок» на верх сайдбара и добавить настройки, которые есть в
Streamlit-аннотаторе hakaton_nornikel, но отсутствуют в React-корректоре shlif-web:
блок «Вид» (прозрачность маски, яркость, CLAHE-фильтр) и блок «Слои-подсказки»
(тёмные сегменты, % матрицы — превью → применить в маску). Настройки должны
показываться в зависимости от активного инструмента, а не все сразу.

Решения (подтверждены пользователем):
- Сайдбар в режиме крупного плана делится на два таба: **«Редактирование»** (по
  умолчанию/активный) и **«Отчёт»** — вместо одного длинного списка карточек.
- Блок «Тёмные сегменты» — это слой-подсказка (превью + кнопка «Применить»), а не
  отдельный кликабельный инструмент, как сейчас (`threshold`-тул убирается).

## Что меняем

### Таб-сайдбар (`components/corrector/Corrector.tsx`)
- Локальный `useState<"edit" | "report">("edit")`.
- Переключатель — существующий класс `.seg` (тот же вид, что и переключатель
  режимов в топбаре), два пункта: «Редактирование» / «Отчёт».
- Таб «Редактирование»: карточка «Редактор масок» целиком (инструмент, кисть,
  слои, вид, слои-подсказки, undo/redo/save) — это то, что видно сразу, без
  прокрутки мимо карточки отчёта. Таб «Отчёт»: текущий проп `info` без изменений
  (Образец / Вердикт / скачать PDF).
- Пока активен таб «Отчёт», canvas переходит в pan-only: `onPointerDown`
  игнорирует brush/eraser/superpixel/threshold-логику независимо от
  `state.tool` (сохраняем поведение zoom/pan). Это реализация принципа «Вид не
  меняет маску» на уровне таба — просмотр отчёта не может случайно задеть маску.

### Инструмент — настройки по режиму
- Список инструментов: Кисть, Ластик, Суперпиксель, Рука (Тёмные области —
  убираем, см. «Слои-подсказки» ниже).
- Слайдер размера кисти показывается **только** для Кисть/Ластик (сейчас
  показан всегда, независимо от инструмента).
- Новое: когда `tool === "superpixel"`, показывается сегмент-переключатель
  «Ставить/Убирать» (локальный `useState`, по умолчанию «Ставить», не в
  reducer — это настройка инструмента, а не данные маски). Сейчас клик по
  суперпикселю всегда устанавливает слой; «Убирать» должно сбрасывать в
  matrix (класс 0) для фазовых слоёв или гасить бит talc.

### Вид (не меняет маску) — новый блок, виден всегда в табе «Редактирование»
- **Прозрачность маски**: слайдер 0.10–0.90, шаг 0.05, по умолчанию 0.50.
  Заменяет захардкоженные веса подмешивания в `composePixel`
  (0.45/0.55 для фаз, 0.4/0.6 для talc) на один `maskAlpha`, общий для всех
  слоёв: `out = (1 - maskAlpha) * base + maskAlpha * color`.
- **Яркость**: слайдер 0.40–2.60, шаг 0.1, по умолчанию **1.00** (нейтрально —
  текущий вид не меняется, пока пользователь не тронет слайдер).
- **Фильтр — CLAHE (контраст)**: тумблер, по умолчанию выключен. Реализация —
  `lib/mask/enhance.ts`: яркость (поэлементное умножение) + CLAHE по каналу
  яркости (luma), без перехода в Lab: считаем `Y = 0.299R+0.587G+0.114B`,
  тайловое CLAHE (тайлы 8×8, клип-лимит 2.5 — те же параметры, что и
  `cv2.createCLAHE(clipLimit=2.5, tileGridSize=(8,8))` в nornikel) по Y,
  восстанавливаем RGB через коэффициент `Y'/max(Y,1)` на каждый канал.
  Пересчитывается в закэшированный буфер только при смене яркости/фильтра (не
  на каждый мазок кисти — как `_enhanced()` в nornikel).
- В `Corrector.tsx` `composePixel` сейчас читает базовый пиксель напрямую из
  `baseRGBA.current`. Вводим промежуточный `enhancedRGBA` ref: при
  яркости=1.0 и выключенном CLAHE это копия `baseRGBA`; `composePixel`
  переключается на чтение из `enhancedRGBA`, пересчитываемого эффектом на
  `[brightness, filt]` (полная перерисовка канваса, как и при смене
  видимости слоёв).
- **Размер на экране, px — не добавляем.** В приложении уже есть полноценный
  zoom/pan (колесо + кнопки), отдельный слайдер ширины дублировал бы эту
  функцию и путал бы пользователя.

### Слои-подсказки (превью → применить в маску) — новый блок
- **Тёмные сегменты, % матрицы**: слайдер 10–70, шаг 5, по умолчанию 45.
- Вычисление на клиенте (`lib/mask/darkpercent.ts`): 256-бинная гистограмма
  карты темноты (`mapUrl(jobId, "darkness")`, уже загружается) по пикселям
  матрицы (`phaseMap === 0`), префиксная сумма → мгновенный поиск порога по
  перцентилю. В отличие от nornikel (ранжирует целые суперпиксели), считаем
  попиксельно — проще и не менее точно для этой подсказки.
- **«Показать превью»**: тумблер, лиловая подсветка (тот же тон, что и
  `DARK_RGB` в nornikel) поверх подходящих пикселей, которые ещё не в маске.
- **«+ Применить к тальку»**: кнопка, OR-ит вычисленную маску в `state.talc`
  через существующий `applyTalc` (попадает в undo-стек как обычная правка).
  Применяется всегда к слою тальк, независимо от того, какой слой сейчас
  выбран для рисования — как в nornikel.

### Новые файлы
- `lib/mask/enhance.ts` — чистые функции яркости + CLAHE-по-luma над
  `Uint8ClampedArray` RGBA-буфером. Без DOM/canvas API — тестируется как
  `reducer.ts` (`node:test`, прямой импорт `.ts`).
- `lib/mask/darkpercent.ts` — гистограмма/перцентиль-порог + построение
  булевой маски.

### CSS (`app/globals.css`)
- `.switch` — компактный тумблер (вкл/выкл) для CLAHE и «Показать превью»,
  на токенах `--brand`/`--surface-3`/`--border` (нового цветового языка не
  вводим).

## Принципы, которым следуем
- Значения по умолчанию не меняют текущий вид (яркость 1.0, CLAHE выкл.) —
  добавляем возможность, а не меняем поведение по умолчанию.
- «Слои-подсказки» — превью, затем осознанное применение кнопкой, а не
  мгновенный клик по холсту (сейчас так у `threshold`-тула) — то же самое
  UX-решение, что уже описано и подтверждено в диалоге.
- Логика canvas/рисования (`onPointerDown`/`onPointerMove`/composePixel-цикл)
  не переписывается с нуля — расширяется параметрами (maskAlpha, enhanced
  base), сам pixel pipeline (rAF-батчинг, dirty-indices) сохраняется.

## Вне объёма (YAGNI)
- «Размер на экране, px» — см. обоснование выше.
- Тумблеры «показать маску / показать сетку суперпикселей / показать оригинал
  рядом», U-Net/SAM-подсказки — есть в nornikel, но не были запрошены и не
  имеют аналога/данных в текущем API (нет отдельной ручки для показа исходника
  без правок, нет SLIC-границ в виде готового слоя, нет U-Net/SAM превью в
  этом сервисе).
- Таб «Отчёт» в панораме (`PanoramaWorkspace.tsx`) — там и так только
  просмотр, править нечего, табы не нужны.

## Проверка
- `npm test` — существующие тесты (`reducer`/`client`/`encode`) остаются
  зелёными; новые тесты на `enhance.ts` и `darkpercent.ts` в том же стиле.
- `npm run build` — typecheck + сборка без ошибок.
- Ручная проверка в браузере (`next dev`) на реальной задаче в режиме
  «Крупный план»: переключение табов, инструмент → настройки скрываются/
  показываются по режиму, слайдеры вида меняют картинку без изменения маски,
  превью тёмных сегментов → применить → появляется в маске и в undo.
