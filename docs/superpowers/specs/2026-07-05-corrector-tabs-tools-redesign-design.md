# Редактор масок: табы «Отчёт/Редактирование» + новая панель инструментов (макет 2a)

**Дата:** 2026-07-05
**Референс:** `Редактор масок.dc.html`, фрагмент `#2a` («Табы · Инструмент · Размер кисти»)

## Цель

Перенести дизайн фрагмента 2a из макета в реальный `Corrector.tsx`: сайдбар редактора
получает настоящее переключение «Отчёт» / «Редактирование» вместо одновременного показа
инфо-блока и инструментов, а сам блок инструментов получает плитки 2×2 с иконкой,
подписью и бейджем горячей клавиши, и отдельный блок «Размер кисти» со слайдером и
кнопками-пресетами. Логика рисования/canvas/reducer не меняется — только сайдбар.

Решения (подтверждены пользователем):
- Табы — настоящее переключение контента, не декоративные.
- Инструмент «Тёмные области» (threshold) в сетку 2×2 не входит — остаётся отдельным
  контролом под сеткой, без плитки и без горячей клавиши, ровно как сегодня.
- Перестилизуем только то, что показывает фрагмент 2a (табы, сетка инструментов, блок
  кисти). Легенда слоёв и кнопки undo/redo/save остаются как есть.
- Горячие клавиши (B/E/S/H) и пресеты кисти (4/12/32 px) — рабочие, не просто визуал.

## Что меняем

### `components/corrector/Corrector.tsx`
- Новое состояние `tab: 'report' | 'edit'` (по умолчанию `'edit'`).
- В `.ws-side` первым элементом — полноширинный `.seg` (переиспользуем существующий класс
  сегмент-контрола, визуально это и есть табы из макета) с кнопками «Отчёт» / «Редактирование».
- `tab === 'report'` → показываем сегодняшний `{info}` без изменений (карточка образца,
  `VerdictPanel`, ссылка на PDF).
- `tab === 'edit'` → один `.card` с новым содержимым `side-b` (без заголовка
  «Редактор масок» — табы уже дают контекст):
  - **Инструмент** — `.tool-tiles` (grid 2×2): Кисть (B) / Ластик (E) / Суперпиксель (S) /
    Рука (H). Активная плитка — акцентный фон/рамка/цвет (как в макете).
  - **Тёмные области (threshold)** — не в сетке; текущий инлайн-слайдер `.ctl` показывается,
    только если выбран этот инструмент (переключается не плиткой, а иначе — см. ниже).
  - **Размер кисти** — заголовок с моно-значением справа, полноширинный `.slider`, под ним
    ряд из 3 кнопок-пресетов (4/12/32 px).
  - Легенда слоёв (`.layers`) и блок undo/redo/save — без изменений, просто ниже.
- Инструмент «Тёмные области» не имеет своей плитки в сетке 2×2 (сетка — строго 4 плитки
  из макета). Доступ к нему: пятая кнопка-ссылка под сеткой (`.tool-more`, текстовая,
  без иконки-плитки), по клику переключает `state.tool = 'threshold'` — как и раньше, слайдер
  порога появляется под ней при активном инструменте.
- Обработчик `keydown` на уровне компонента: клавиши B/E/S/H переключают инструмент, только
  когда `tab === 'edit'` и не зажат ни один модификатор (Ctrl/Meta/Alt).
- Клик по пресету кисти напрямую ставит `state.brush`; активная кнопка подсвечивается.

### `components/icons.tsx`
- Новые иконки `IconEraser` и `IconGrid` (superpixel) — SVG-пути 1:1 из фрагмента 2a
  макета, тот же стиль (`stroke-width 1.8`, `viewBox 24`), что и остальные иконки файла.

### `app/globals.css`
Новые классы, использующие существующие токены (`--brand*`, `--surface*`, `--border*`,
`--font-mono`), без сырых `oklch()`-значений из макета:
- `.tool-tiles` — grid `1fr 1fr`, gap как у `.tool-group`.
- `.tool-tile` (+ `.active`) — плитка: иконка, подпись, `kbd`-бейдж в углу.
- `.tool-tile kbd` — моно-шрифт, маленький бейдж на поверхности `--surface-2`.
- `.brush-presets` / `.brush-preset` (+ `.active`) — ряд из 3 равных кнопок с моно-цифрой.
- `.tool-more` — текстовая кнопка-ссылка под сеткой для порога.

## Принципы (из дизайн-системы «Шлиф»), которым следуем

- Никаких новых сырых `oklch()` — только существующие CSS-переменные из `globals.css`.
- Латунный акцент (`--brand*`) — единственный цвет выделения активного состояния.
- Граница вместо тени, приборная форма (radius как у существующих `.tool-group`/`.card`).
- Только line-иконки stroke 1.8, никаких эмодзи.

## Вне объёма (YAGNI)

- Полная перестилизация всего рельса под 1a/1b (легенда слоёв, undo/redo/save) — только
  блок 2a.
- Изменение логики холста, reducer, undo/redo, сохранения масок.
- Горячая клавиша для «Тёмные области» — в макете 2a её нет, не придумываем.
