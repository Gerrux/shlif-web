# Редактор масок: панель инструментов по макету 2a (плитки + пресеты кисти)

**Дата:** 2026-07-05
**Референс:** `Редактор масок.dc.html`, фрагмент `#2a` («Табы · Инструмент · Размер кисти»)

## Цель

Перенести дизайн фрагмента 2a из макета в реальный `Corrector.tsx`.

**Уточнение по ходу работы:** на момент брейнсторминга локальный чекаут был на 118
коммитов позади `origin/master`. В актуальном коде (worktree от `origin/master`) табы
«Отчёт» / «Редактирование» уже реализованы по-настоящему (`sideTab` state, переключает
`{info}` ↔ карточку редактора) — этого делать не нужно. Инструмент «Тёмные области»
(threshold) тоже уже убран из списка инструментов: сейчас `TOOLS` содержит ровно 4
записи — Кисть/Ластик/Суперпиксель/Рука, один в один с сеткой макета 2a. Порог остался
как отдельная непрерывная фича «Слои-подсказки» (превью тёмных сегментов + кнопка
«Применить к тальку»), это не инструмент и вне объёма данной задачи.

Реальный объём: только визуал блока «Инструмент» (сделать плитками 2×2 с иконкой,
подписью и бейджем горячей клавиши) и блока «Размер кисти» (шапка со значением, слайдер,
кнопки-пресеты) — плюс рабочие горячие клавиши и клики по пресетам. Всё остальное в
`Corrector.tsx` (табы, слои, вид/яркость/CLAHE, слои-подсказки, undo/redo/save) не
трогаем.

Решения (подтверждены пользователем на момент брейнсторминга, actual к текущему коду):
- Перестилизуем только то, что показывает фрагмент 2a: сетка инструментов + блок кисти.
- Горячие клавиши (B/E/S/H) и пресеты кисти (4/12/32 px) — рабочие, не просто визуал.

## Что меняем

### `components/corrector/Corrector.tsx`
- Блок «Инструмент» (сейчас — горизонтальный `.seg` с `TOOLS.map`) заменяется на
  `.tool-tiles` — grid 2×2. Каждая плитка: иконка + подпись + `kbd`-бейдж горячей клавиши
  в углу (B — Кисть, E — Ластик, S — Суперпиксель, H — Рука). Активная плитка — акцентный
  фон/рамка (латунь), как остальные активные состояния в приложении.
- Блок «Размер кисти» (сейчас — `.ctl` инлайн с ползунком и числом), показывается только
  при `state.tool === "brush" || "eraser"` как и сегодня, получает: заголовок с
  моно-значением справа (`{brush}px`), полноширинный `.slider`, и под ним ряд из 3-х
  кнопок-пресетов 4/12/32 (клик → `setState({ ...state, brush: v })`, активная кнопка —
  та, что равна текущему `state.brush`).
- Новый `useEffect` с `window.addEventListener("keydown", …)`: клавиши b/e/s/h (без
  учёта регистра) переключают `state.tool`, только когда `sideTab === "edit"`, ни один
  из модификаторов (ctrlKey/metaKey/altKey) не зажат, и фокус не на текстовом поле
  (`e.target` не `input`/`textarea`/`[contenteditable]` — в этой панели таких нет, но
  проверка на будущее). Слушатель вешается один раз и снимается при размонтировании;
  актуальные `state`/`sideTab` читаются через рефы (как уже сделано для `toolRef`,
  `sideTabRef` в файле), чтобы не пересоздавать обработчик на каждый рендер.

### `components/icons.tsx`
- Уже добавлены `IconEraser` и `IconGrid` (superpixel) — SVG-пути 1:1 из фрагмента 2a
  макета, тот же стиль (`stroke-width 1.8`, `viewBox 24`), что и остальные иконки файла.

### `app/globals.css`
Новые классы в секции «Панель инструментов корректора», на существующих токенах
(`--brand*`, `--surface*`, `--border*`, `--font-mono`), без сырых `oklch()`:
- `.tool-tiles` — `display:grid; grid-template-columns:1fr 1fr; gap:8px`.
- `.tool-tile` (+ `.active`) — плитка-кнопка: рамка `--border`/`--brand` в active,
  фон `--surface`/`--brand-tint` в active, текст `--text-2`/`--brand-ink` в active.
- `.tool-tile kbd` — маленький моно-бейдж в углу плитки, фон `--surface-2`.
- `.brush-head` — флекс-строка заголовок + моно-значение (аналог `.side-h .ann`).
- `.brush-presets` / `.brush-preset` (+ `.active`) — ряд из 3 равных кнопок с моно-цифрой.

## Принципы (из дизайн-системы «Шлиф»), которым следуем

- Никаких новых сырых `oklch()` — только существующие CSS-переменные из `globals.css`.
- Латунный акцент (`--brand*`) — единственный цвет выделения активного состояния.
- Граница вместо тени, приборная форма (radius как у существующих `.tool-group`/`.card`).
- Только line-иконки stroke 1.8, никаких эмодзи.

## Вне объёма (YAGNI)

- Табы «Отчёт»/«Редактирование» — уже реализованы, не трогаем.
- «Слои-подсказки» (порог тёмных сегментов), «Вид» (прозрачность/яркость/CLAHE), легенда
  слоёв, undo/redo/save — без изменений.
- Изменение логики холста, reducer, сохранения масок.
- Горячая клавиша для «Слои-подсказки» — в макете 2a её нет, не придумываем.
